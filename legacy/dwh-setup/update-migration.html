Migration von Debian 8 und CentOS 7

Für die Migration von älteren AKTIN Data Warehouses der Betriebssysteme Debian 8 und CentOS 7 auf Ubuntu Server 20.04 LTS wird ein Migrationsskript zur Verfügung gestellt. Bitte beachten Sie, dass über dieses Migrationsskript nur die Version 1.4.1 installiert wird. Um die aktuellste Version des AKTIN Data Warehouse mit Debian-Paketierung zu installieren, müssen Sie anschließend noch eine Migration zu der aktuellen AKTIN Version durchführen. Folgen Sie hierzu den Hinweisen auf Update von DWH Version 1.3 oder 1.4 Wenn Sie ein anderes Betriebssystem mit einer älteren Version des AKTIN Data Warehouses verwenden und migrieren möchten, kontaktieren Sie den AKTIN IT-Support.

Anmelden als root

Für die Ausführung ist wichtig, dass der Nutzer als root angemeldet ist. Sollten Sie den Nutzer root noch nicht aktiviert haben, so folgen Sie bitte den Anweisungen im Abschnitt Erste Schritte des Installationsskriptes. Alle folgenden Befehle sollten nur als Nutzer root ausgeführt werden.

Durchführung der Migration

1. Datensicherung

Für die Durchführung der Migration muss das Migrationspaket heruntergeladen und entpackt werden. Haben Sie direkt vom Server aus die Möglichkeit, das Paket herunterzuladen, können Sie hierfür den folgenden Befehl nutzen:

wget https://www.aktin.org/software/repo/org/aktin/dwh/dwh-migration/1.4.1/dwh-migration-1.4.1.tar.gz

Besteht diese Möglichkeit nicht, können Sie das Paket alternativ über diesen Link von einem anderen Rechner aus herunterladen.

Kopieren oder laden Sie anschließend die Datei ins /root-Verzeichnis Ihres Servers. Entpacken Sie diese Datei und wechseln Sie anschließend in das Verzeichnis des Updates:

cd /root

tar xvzf dwh-migration-1.4.1.tar.gz

cd dwh-migration

Der Inhalt des Ordners (über den Befehl ls) sollte in etwa wie folgt aussehen:

dwh-migration
- packages/
- aktin_apply_backup.sh
- aktin_create_backup.sh

Markieren Sie die enthaltenen Skript gegebenenfalls als ausführbar und führen Sie anschließen das Skript aktin_create_backup.sh aus:

chmod +x *.sh

./aktin_create_backup.sh

Das Skript wird nun eine komprimierte Datensicherung folgender Komponenten erstellen:

  • Die Datei aktin.properties
  • Die Datenbanken i2b2 und aktin
  • Den Ordner /var/lib/aktin

Sollten Sie noch weitere Dateien absichern wollen, melden Sie dies bitte an den AKTIN IT-Support. Notieren Sie sich außerdem manuell den Inhalt der Datei email.config. Die dort enthaltenen Informationen Ihres E-Mail-Servers müssen anschließend in die aktuelle Version der aktin.properties eingegeben werden. Sichern Sie zum Schluss den gesamten Ordner dwh-migration auf einem externem Medium (beispielsweise auf einem USB-Stick).

2. Installation des neuen Betriebssystems

Überschreiben Sie das Betriebssystem Ihres Servers mit Ubuntu Server 20.04 LTS oder einem ähnlichen Betriebssystem. Eine Anleitung für die Installation von Ubuntu Server finden Sie auf der Seite Server-Installation.

3. Einspielen der Datensicherung

Kopieren Sie nach erfolgreicher Installation alle gesicherten Dateien des externen Mediums in einen beliebigen Ordner des neu installierten Betriebssystems. Führen Sie in diesem Ordner das Skript aktin_apply_backup.sh aus. Markieren Sie es gegebenenfalls als ausführbar:

chmod +x *.sh

./aktin_apply_backup.sh

Das Skript wird zunächst automatisch das AKTIN-Installationsskript und das AKTIN-Updateskript ausführen, um die Version 1.4.1 des AKTIN Data Warehouse auf dem neuen Betriebssystem zu installieren. Anschließend werden Ihre gesicherten Dateien in das neu installierte Data Warehouse eingespielt.

Die standardmäßige Datei aktin.properties wird dabei zum größten Teil mit dem Inhalt Ihrer abgesicherten aktin.properties überschrieben. Es ist darauf hinzuweisen, dass in aktin.properties einige Felder für die Konfiguration des E-Mail-Servers hinzugekommen sind. Tragen Sie dort Ihre notierten Informationen der Datei email.config ein. Stellen Sie außerdem auch sicher, dass das Feld

email.session=local

entsprechend bezeichnet ist. Nur mit dem Keyword local wird die Konfiguration innerhalb der aktin.properties genutzt. Beachten Sie außerdem auch die Hinweise im Abschnitt Konfiguration und lokale Einstellungen des Installationsskriptes.

Die aktin.properties können Sie über den Befehl nano manuell bearbeiten. Starten Sie nach Abschluss der Bearbeitung den WildFly-Server neu, um die neue Konfiguration zu laden:

nano /opt/wildfly/standalone/configuration/aktin.properties

service wildfly restart

Nach erfolgreichen Neustart können Sie auf der Seite http://<IHR SERVER>/aktin/admin/plain/test.html die Grundfunktionalität testen. Eine weiterführende Anleitung zum Testen der Verbindung finden Sie im Bereich Test der Betriebsfähigkeit.
